# Implement Free Cloud-Based Uptime Monitor for Raspberry Pi

## Goal

Set up a free UptimeRobot account to monitor your Pi's publicly accessible analytics server and send email alerts to `shaivure@gmail.com` if it becomes unreachable.

## Pi URL

Analytics server: `https://raspberrypi.tailc23afa.ts.net:8443`

## Implementation Steps

### Step 1: Create UptimeRobot Account

1. Go to https://uptimerobot.com
2. Sign up with `shaivure@gmail.com` (free tier)
3. Free tier: 50 monitors, 5-minute check intervals, email alerts

### Step 2: Add Monitors

Create these monitors in the UptimeRobot dashboard:

| # | Monitor Name | Type | URL | Interval | Purpose |
|---|---|---|---|---|---|
| 1 | BBC Radio Pi - Nginx (HTTPS) | HTTPS | `https://raspberrypi.tailc23afa.ts.net:8443` | 5 min | Full stack check (reverse proxy + SSL) |
| 2 | BBC Radio Pi - Search Server | HTTP | `http://raspberrypi.tailc23afa.ts.net:5001/health` | 5 min | Search service health endpoint |
| 3 | BBC Radio Pi - Analytics Server | HTTP | `http://raspberrypi.tailc23afa.ts.net:5002/stats` | 5 min | Analytics service responding |
| 4 | BBC Radio Pi - Ping | Ping | `raspberrypi.tailc23afa.ts.net` | 5 min | Network-level reachability |

**Monitor configuration:**
- **HTTP/HTTPS monitors:** Check for HTTP 200 status, 30s timeout
- **Ping monitor:** Default ICMP, 30s timeout
- **Alert contact:** `shaivure@gmail.com` for all monitors
- **Alert sensitivity:** 1 consecutive failure (fastest alert) or 2 (reduces false positives)

### Step 3: Configure Email Alerts

1. Dashboard → My Settings → Alert Contacts
2. Verify `shaivure@gmail.com` is the active alert contact
3. Enable email notifications for both "Down" and "Up" events
4. Optional: Enable SMS alerts (limited on free tier)

### Step 4: Create Project Documentation

Create `docs/monitoring.md` to record:
- UptimeRobot account details
- Monitor list with URLs and settings
- How to verify monitors are working
- How to pause monitoring during maintenance

### Step 5: Verify

1. Confirm all monitors show "Up" in UptimeRobot dashboard
2. Test alert: stop the search service (`sudo systemctl stop bbc-radio-search.service`) and confirm email arrives
3. Restart service and confirm "back up" notification arrives
4. Check `shaivure@gmail.com` inbox (and spam folder)

## Notes

- Port 8443 (Nginx) uses self-signed SSL — UptimeRobot may report SSL warnings. If so, disable SSL verification for that monitor or use the HTTP monitors (ports 5001/5002) as the primary monitors.
- The search server `/health` endpoint is the most reliable single indicator since it's a dedicated health check with no dependencies beyond the service itself.
